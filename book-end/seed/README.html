使用自己的seed项目

  如果不是单页应用，或者在官网中找不到合适的项目结构，那么可以自己搭建seed项目；

  1. 创建目录generator-name,目录名必须符合generator-xxx的结构；
  2. npm init初始化node模块，生成package.json文件；
    {
"name": "generator-name",
"version": "0.1.0",
"description": "",
"files": [
 "app",
 "router"
],
"keywords": ["yeoman-generator"],
"dependencies": {
 "yeoman-generator": "^0.17.3"
}
}

  3. 构造如下目录结构：

    ├───package.json
├───app/
│   └───index.js
└───router/
 └───index.js

    app目录下的index.js是初始化项目的node代码，在运行yo name的时候， 会执行这个初始化文件；router下的index.js是实现形如yo name:router foo这样的命令，例如在generator-angular中可以使用yo angular:controller user命令生成一个controller文件，这个文件包含基本的controller结构，无需每次自己写；

  4. 运行sudo npm link，在node全局模块目录下建立该模块的快捷方式，这样就可以在任何地方运行yo name了；

发布Nodejs模块

  要发布自己的generator到npm服务器上，你需要有社区帐号，你只需执行命令npm adduser，然后根据提示输入你的用户名密码等信息即可创建帐号，在你的generator目录下执行npm publish，没有异常的话又是一阵刷屏，然后就提示你发布成功，你可以到npm官网上搜到你发布的模块了。发布后别人就可以跟使用其他node模块一样使用你的generator啦。
